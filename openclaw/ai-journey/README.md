# AI Journey

A structured path to becoming an AI-augmented engineer.

## Why This Journey?

The software engineering landscape is changing. AI can write code, but:
- It needs good prompts
- It can't debug production issues
- It doesn't understand your business
- It can't deploy to customer sites

This journey builds skills that complement AI, not compete with it.

## The Framework

```
AI-Augmented Engineer = Problem Solving + AI Collaboration + Domain Knowledge + Communication
```

### Daily Practice
- Use AI to help, not replace thinking
- Verify everything AI produces
- Build end-to-end projects
- Document what you learn

## Structure

- `notes/` - Daily learnings, insights
- `projects/` - Hands-on projects
- `skills/` - Skill tracking
- `resources/` - Learning resources

## Start Here

1. Read `skills/` to see current level
2. Pick a project from `projects/`
3. Take notes in `notes/`
4. Track progress weekly
